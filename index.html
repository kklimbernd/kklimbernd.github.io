軟體工程師扶持計畫 – 申請者 Kim

兩百字內的個人簡介

Hello 我是Kim，畢業自淡江英文系，在校期間積極參與戲劇社團，畢業後在影視產業做節目企劃，目前已經離職，除了轉換領域外，也考慮報考資管系研究所。

我會用兩個形容詞形容自己：

  • 樂於嘗試：

    • 不害怕挑戰舒適圈，也不會因為非本科系而卻步——大學時因為喜歡視覺、平面設計，而自學了相關軟體。對我來說，走出舒適圈可以看見自己更多可能性。

  • 熱愛實作：

    • 我很喜歡落實想像、完成作品的過程，如同過去所經歷的戲劇創作、節目製作、平面設計等。在轉換跑道之際，目前希望能以程式能力作為發展目標。

為什麼想成為工程師
( 請以自我期許帶入：前端、後端、或全端工程師 )

過去熟悉的領域（影視產業、藝文領域）較為傳統，每個專案初期都會有大量重複的資料搜集工作。每當新專案開始時，就會耗費很多時間在例行的事務上，因此使我萌生對程式撰寫的好奇心，希望透過自動化減輕工作負荷。

參與創作的過程中，我發現了藝術結合科技勢必為未來的娛樂趨勢，希望透過訓練營熟悉程式，試著找到自己能否在這個新的結合產業有所發展。

以網頁製作來說，以我過去平面設計的經驗及興趣，可能比較適合前端！但是如上一點所述，我想也會需要有後端的知識，所以目前希望朝全端發展。

為了成為工程師，曾經做過什麼努力？

  • 線上課程：

    • 過去是文組學生，為了確認自己是否能夠理解相關理論，所以有上了一些課程，目前都還可以跟上。

    1. CS50’s Introduction to Computer Science：Harvard X 線上課程。

    2. 大學開放式課程：計算機概論、資料結構等。

  • Python 爬蟲實作

    • 剛好兼職的工作需要特定網頁的數十頁資料。為了測試自己是否能夠學習程式，我大量參考彭彭老師的頻道、python相關書籍、Stack Overflow。最後勉強將網頁內容爬下來後，內心成就感爆棚，也因此下定決心要繼續努力。

對於工程師的職業想像是什麼？

我覺得工程師的工作內容主要是以程式語言來解決數位生活的問題。

  1. 專案實作型工作模式

    2. 專案執行後能有具體結果。

  2. 職業發展區域性限制較低、工作機會多

    3. 以南部來說，節目企劃、行銷等的工作機會都遠比工程師工作機會少，且薪資水平低於我的預期，這也是我選擇轉換領域的原因

  3. 網路社群多

    4. 生態完整，網路上有很多相關社群，接受產業新知和技術升級等都相對容易。

  4. 可結合性高

    5. 如題2所答，儘管沒有成為純工程師，將程式能力結合過去所學，都有機會有驚喜的結果。

如果參與這個計畫，會怎麼安排學習時間？

目前的兼職工作下班時間固定，週間晚上都至少可以安排三小時的學習時間，扣除訓練內容消化以及外務，一週應可有30~35小時可供運用。

如果參與這個計畫，預期會碰到什麼困難？你打算怎麼解決它？

目前可預期最大的困難會是：全部！（哈哈哈哈）

以過去學習外語、繪圖軟體的經驗來說，我覺得了解理論後再加上大量實作熟悉，就可以慢慢上手。理論知識補充除了上網搜尋相關之外，我會使用相關書籍。書籍和程式、套件本身的Documentation，脈絡較完整，比較有效解決問題。
